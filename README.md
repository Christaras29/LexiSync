# LexiSync: ER (Entity Resolution) Matching System

Το **LexiSync** είναι ένα σύστημα Ταυτοποίησης Οντοτήτων (Entity Resolution) σχεδιασμένο για την αντιστοίχιση δεδομένων.

## Δομή Αρχείων

📂 Archive/
📁 data/ — Φάκελος αποθήκευσης των συνόλων δεδομένων (datasets)
📁 results/ — Τα αποτελέσματα από όλα τα πειράματα
📄 lexi_sync.py — Η κεντρική κλάση του συστήματος και η ροή εκτέλεσης
📄 trainer.py — Custom trainer για την εκπαίδευση του συστήματος
📄 blocker.py — Κλάση για τη δημιουργία του ANNS Blocker (με τη χρήση FAISS)
📄 ablation.py — Εκτέλεση του Ablation Study για τον μειωτικό συντελεστή w1
📄 bi_encoder.py — Υλοποίηση και εκτέλεση του Bi-Encoder
📄 timing_benchmark.py — Script για την ανάλυση κόστους/χρόνου
📄 requirements.txt — Οι απαραίτητες βιβλιοθήκες και εξαρτήσεις

## Προαπαιτούμενα & Εγκατάσταση (Requirements)
- Αποσυμπίεση του Archive.
- Έκδοση Python: 3.11.7
- Όλες οι απαραίτητες βιβλιοθήκες και εξαρτήσεις είναι καταγεγραμμένες στο αρχείο requirements.txt.
- Για να εγκαταστήσετε τις απαιτούμενες βιβλιοθήκες, εκτελέστε την παρακάτω εντολή στο τερματικό σας: pip install -r requirements.txt

## Ανάλυση Components
1. Πυρήνας του Συστήματος
- lexi_sync.py: Αποτελεί την κεντρική κλάση του συστήματος. Περιλαμβάνει τη βασική λογική και τη διαδικασία εκτέλεσης της ταυτοποίησης.
- trainer.py: Υλοποιεί τον custom trainer που είναι υπεύθυνος για τη διαδικασία εκπαίδευσης του μοντέλου.
2. Μηχανισμός Blocking
- blocker.py: Περιέχει την κλάση του ANNS (Approximate Nearest Neighbors Search) Blocker, ο οποίος αξιοποιεί τη βιβλιοθήκη FAISS για την αποδοτική μείωση του χώρου αναζήτησης ζευγών.
3. Πειράματα & Αξιολόγηση 
- Ablation Study (ablation.py): Πραγματοποιεί τη μελέτη ablation study με σκοπό την τεκμηρίωση της επίδρασης του μειωτικού συντελεστή w1. Λειτουργεί ως επέκταση (extend) της κλάσης lexi_sync ώστε να διασφαλίζεται η πανομοιότυπη εκτέλεση.
- Bi-Encoder (bi_encoder.py): Εκτελεί την λογική του Bi-Encoder.
- Cost Analysis (timing_benchmark.py): Εκτελεί την ανάλυση κόστους/χρόνου της έρευνας.
